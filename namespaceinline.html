Detailed Description

The inline.py file contains functions for generating inline keyboards for various operations. 
get_file_menu() creates an inline keyboard for file operations, such as upload and delete. 
get_inline_doc_keyboard() generates a keyboard for document operations, such as viewing columns 
and document information. get_inline_doc_columns_keyboard() creates a keyboard for column operations, 
such as selecting columns and data types. get_return_keyboard() generates a keyboard for returning to previous menus.

◆ get_file_menu()

inline.get_file_menu ( ) 
This function generates an inline keyboard for file operations within a chat interface.

Behavior:
 - Attempts to retrieve the list of files in the user’s data folder (associated with UserData.id).
 - If no files are found, it adds a button labeled “Upload File” to the keyboard.
 - If files exist, it adds buttons for each file (with their names) and a delete button (❌) for each file.
 - The layout of the keyboard is adjusted to display two buttons per row.
 - The resulting inline keyboard markup is returned.

◆ get_inline_doc_columns_keyboard()

inline.get_inline_doc_columns_keyboard ( ) 
This function generates an inline keyboard specifically designed for column-related operations 
within a user interface.

Behavior:
 - The purpose of this function is to create an inline keyboard for interacting with document columns.
 - This function does not take any parameters.
 - For each column key-value pair obtained from the columns_print(DocInfo.name) function:
     * A button is added to the keyboard with the format: "Column Key -- Column Value".
     * The callback_data parameter is set to include the column name and type
 - The adjust(1) method is called to adjust the layout of the keyboard, ensuring a 
    single column of buttons.
 - The function returns the generated inline keyboard markup as an InlineKeyboardMarkup object.

inline.get_inline_doc_keyboard ( ) 
This function creates an inline keyboard specifically designed for document-related operations within 
a user interface. It takes no arguments and returns an InlineKeyboardMarkup object, which can be used 
to display buttons for various document-related actions

Behavior:
 - The purpose of this function is to generate an inline keyboard for document-related interactions
 - This function does not take any parameters.
 - “Show Columns”: When clicked, this button triggers the display of document columns.
 - “Show Description”: Clicking this button reveals the document description.
 - “Show Info”: This button provides additional document information.
 - The adjust(3) method is called to adjust the layout of the keyboard. It ensures that 
    the buttons are displayed in a visually appealing manner.
 - The function returns the generated inline keyboard markup as an InlineKeyboardMarkup object.

◆ get_return_keyboard()

inline.get_return_keyboard ( ) 
This function generates an inline keyboard specifically designed for returning to previous menus within a user interface.

Behavior:
 - The purpose of this function is to create an inline keyboard for navigating back to previous menus.
 - This function does not take any parameters.
 - Two buttons are added to the keyboard:
    “Back to Columns”: When clicked, this button takes the user back to the column menu.
    “Back to Files”: Clicking this button returns the user to the files menu.
 - The adjust(2) method is called to adjust the layout of the keyboard, ensuring a visually balanced arrangement of buttons.
