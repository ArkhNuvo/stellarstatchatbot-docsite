Detailed Description

File contains functions to handle callback queries from the user interface. 
It allows users to select files, view column information, and choose statistical 
options for analysis. Additionally, it supports options for displaying descriptive 
statistics and generating various types of plots based on the selected data.

callback.choice_col_option ( CallbackQuery call,
StatInfo callback_data ) 
Interacts with user-selected options for generating descriptive statistics and visualizations for a chosen variable within a dataset.

Parameters:
 - call: An object representing a callback query, containing data about the user's selection.
 - callback_data: Object holding information related to the chosen statistical option for the variable.

Behavior:
 - Retrieves user's choice: Accesses the specific statistical option selected from the callback data.
 - Performs operations based on choice:
    * Quantitative variables:
        >Calculates and presents descriptive statistics (mean, median, mode, range, etc.).
        >Generates various visualizations (distribution plots, box plots, etc.).
    * Categorical variables:
        >Provides counts of unique values, lists unique values, and calculates value counts (percentages).
        >Creates pie charts and count plots.

Returns 
 - Message or image: Sends the appropriate response to the user, either as a formatted text message or a generated image.

callback.chose_file_from_storage ( CallbackQuery call,
DocInfo callback_data ) 
This function handles file selection from storage based on user interactions. The purpose of this 
function is to handle file selection (either selecting or deleting) based on user interactions with inline buttons.

Parameters:
 - call (CallbackQuery): The incoming callback query triggered by the user's action.
 - callback_data (DocInfo): Data received from the callback query, including the file name and the action (select or delete).

Behavior:
 - If the action is 'select', the chosen file name is displayed to the user.
 - If the action is 'delete', the specified file is removed from the storage, and the file count is updated.
 - Appropriate messages are sent to the user based on the action taken (selecting or deleting).
 - If there are no files left, a message informs the user that there are no files to analyze
